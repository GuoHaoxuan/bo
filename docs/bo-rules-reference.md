## 1. 这是什么

波是一个**有节拍的、同步暗出、带资源（气）管理的回合制博弈游戏**。
类似石头剪刀布，但多了「攒气 → 放招」的资源层。可 2 人对决（**波决**）或多人围圈，
玩到只剩一人获胜。核心张力：**攒气 vs. 抢先出手**，以及**跟着节拍做决策的压迫感**。

## 2. 通用机制（跨模式通用，= 引擎要实现的部分）

- **节拍 (beat)**：固定节奏，每拍一回合；所有活人**暗选一个动作**，到拍同步揭示。
  跟不上 / 超时 → **默认出「防」**。
- **气 ＝「按类型分别记的一袋」（不是单一标量！）**：玩家持有 `Map<气类型, 量>`。
  - **气类型 / 资源**：波 / 刀 / 枪 / 屁(原称「气」，改名避免与资源词重名) / …，**各自用对应的「运气」动作单独积累**
    （运气波→波气 +1；100代→波气 +100；运气刀→刀气 +1；运气枪→枪气 +1…）。
  - 攻击 / 合击技消耗**特定类型**的气。波决里只有「波」这一种气，才显得像标量。
  - 可为**小数**（点/推/削波耗 0.1 / 0.5）→ 代码用**定点数 `Qi`**（内部整数，杜绝浮点误差）。
- **代 (generation) ＝ 量级 / 倍率轴**：
  - **只放大「运气量」和「吸取量」**：N代 → 一次运气得 `基础×N` 气（一代波+1、100代波+100）；吸取上限 ＝ 代数。
  - **不放大攻击破防强度；基础防御（防/捂鼻…）也不放大**（恒按 1 代）——但**「学到的」防御技能（如反弹盾）会随代升级**；100代模式下所有防御按 100 代。
  - **代似乎跟人走**（玩家的 prestige 等级；同局可 2代 vs 1代）——❓ 待最终确认。
  - 来一点在 2代+ 改名「**吸**」；吸取 ≤ `min(代数, 对方在运的气量)`。
  - 引擎视角：代 ＝ 每个玩家一个倍率 → **运气 ×代、吸取 ≤代、防御不乘**（除 100代模式）。
- **通道 (channel) ＝ 伤害类型，只决定「被哪种防御反制」**（与「气类型」是**两条独立的轴**）：
  - **物理通道**：波 / 刀 / 枪 三种气的攻击都属物理 → 都被「防」反制。
  - **气通道**：被「捂鼻」反制。
  - 即：**多种气类型可共用一个通道**（波/刀/枪 ＝ 3 种气、1 个物理通道）。
- **合击技 (combo)**：消耗**多种气类型的组合**，产生强力效果。
  - 火龙 = 1波+1刀（**兑掉对方防御**）；火凤 = 1波+1刀+1枪（**穿盾**）；
    刀枪不入 = 1刀+1枪（**对所有通道无限防御**，是个防御性合击）。
  - → 技能的 `cost` 是一个**气类型→量的 bundle**（`Map<气类型,量>`）；简单招单条目、合击多条目；
    防御性合击（刀枪不入）也消耗气，故 `cost` 不是攻击专属；
    某些招消耗「**全部**某类型的气」（下屁释放全部屁气，破防强度＝释放量）；
    还有「**选择型**」消耗——氧气罐 ＝『任意 1 种气 × 1』，由玩家**暗选**哪种。
    → `cost` 三形态：**固定量 / 全部某类型 / 任选 1 种**。
- **防御 (defense)**：**按通道**给出防御力；攻击带**破防强度**与之对比，得出 `防住 / 兑 / 破` 之一。
  - 一般规律：破防强度 `S` vs 防御力 `D` → `S≤D 防住`、`S=D+1 兑`、`S≥D+2 破`；
    **但有的防御没有「兑」档**（如全防）→ 档位需 **per-防御配置**，不能写死。
  - **破防强度**独立于气耗：波系 ＝ 气耗；刀双斩/枪双枪 ＝ 高于气耗（破盾）；下屁 ＝ 本次释放的屁气总量。
    破盾**可被特定防御免疫**（反弹盾免疫双斩/双枪的破盾，对它按面值结算）。
  - 已知防御档位（破防强度 → 结果）：
    - **防**（物理）：2防 3兑 4破
    - **捂鼻**（屁/魔法）：2防 3兑 4破
    - **捂鼻防**（物理 ＋ 屁）：每通道 1防 2兑 3破
    - **全防**（默认禁用·所有通道）：1防 2破（**无兑档**）
    - **刀枪不入**（合击·所有通道）：无限防御
  - 「运气」动作可**自带防御**（大霸 +1）。
  - 光谱：`防住 < 兑 < 破/穿`（火龙 ＝ 把防变成兑；火凤 ＝ 完全穿透）。
- **延迟选择的气债（氧气罐）— 决定：忠实还原「先欠着」**：氧气罐消耗「任意 1 种气 × 1」，玩家**不在出招时指定**扣哪种。
  - **最干净的形式化 ＝「可能气态集合」`S`**：玩家状态不是单个向量，而是一**组**可能的确切气向量。
    - **运气 t (+a)**：`S` 中每个向量的 t 分量 +a。
    - **确定消耗 c**：每个向量 `v → v−c`，付不起的分支**淘汰**；`S` 变空 → **溶**。
    - **氧气罐**：每个 `v` 分裂成 `{ v − 该类型1 : v 里此刻 ≥1 的每种类型 }`（如同时有波/刀 → 分裂成两支）；
      没有任何 ≥1 类型的分支淘汰；`S` 空 → 溶。`[0.5,0.5]` 欠 1 ＝ 无 ≥1 类型 → 不可行。
      → **债只能落在「用罐当下 ≥1 的类型」**；之后才运的气（枪）加到所有分支上，但**抵不了这笔旧账**（分裂发生在过去状态上）。
    - **某招合法 ⟺ `S` 中至少一个向量付得起**（只保留付得起的分支）；全付不起 → 溶。
  - **兑 与债**：发生兑时，防御方清空气 → `S` 塌缩为 `{ 全 0 }`，**债一笔勾销**
    （本就无真实欠款，只是「不知道扣了哪种」的不确定性，清零后即消失）。
  - 仍是**确定性纯函数**（同样的 `S` ＋ 暗选 → 唯一新 `S`，可测）。实现注意：对 `S` **去重/剪枝**防组合膨胀。
  - ⚠️ **反向决定了 `bo_rules` 的气状态表示**：是「**可能气态集合**」而非单个向量/标量，从第一天就这么设计。
- **结算顺序（铁律）**：
  1. **先判溶**：气 < 招式气耗硬放招（或放不存在的招）→ 当场判死，**且其攻击不生效**。
  2. **再判克**：在溶的幸存者之间结算攻击 → 死 / 兑。
- **兑 (duì)**：介于「挡住」与「挡不住」之间的状态；**清空防御方所有气**（如「防」遇 3 气攻击）。
- **目标优先级**（单 / 双体攻击自动锁定目标）：
  `克 ＞ 来一点 ＞ 反弹盾 ＞ 攻击者 ＞ 运气 ＞ 大霸 ＞ …`
  - 来一点靠**强 taunt** 把攻击吸向自己，故排第二。
  - 反弹盾排在攻击者之上，是为了让它有机会把攻击**反弹**回去（弱 taunt）。
  - 已知**克关系**：**自杀 克 来一点**。（激光「互攻按 3」**不是克**，是攻击自带的高互攻强度——见「互攻」条。）
- **互攻**：双方都合法 → **互攻强度高者胜、低者死、等强两边都活**（不是兑）。
  互攻强度一般 ＝ 气耗，但可不同——**激光互攻一律按 3 算**（对所有攻击，不限通道）。
- **夹死**（⚠️ 3 人专用词）：两人攻击把中间一个**正在运气**的人夹死。
- **出局 / 重开**：被判死即出局；**幸存者清空气、继续下一拍**；剩 1 人＝获胜；剩 0 人（如双方同溶）＝平局。
- **明牌 (tell)**：某些招的准备动作会暴露意图（如冲击波运第 6 气 ＝「神龟」❓，明牌）。

### 3.1 魔法通道家族（屁 / 口红 / 扎啤 / 电话 — 平行结构）

4 个魔法通道结构相同：各有**运气 ＋ AOE 攻击**（释放全部、与下屁同型）、满防御、（部分）反弹克制、（部分）强防御。
设计意图：**多通道让防御者猜不准用哪种防**（捂鼻只挡屁、捂嘴只挡口红…）；捂鼻防(物理＋屁)、全防(全通道) 是跨通道妥协。
> 记号：`X反` ＝ 该档**反弹**（挡下并弹回攻击者）；`X防` ＝ 仅挡下。

| 通道 | AOE攻击 | 满防御 2防3兑4破 | **反弹克制 1反2破** | 强防御 6防7破(耗任意气) |
|------|---------|------------------|---------------------|--------------------------|
| **屁**(原「气」) | 下屁 | 捂鼻 | —（无反弹；弃用的掐鼻/捏鼻＝纯防御 防1/防2） | 氧气罐 |
| **口红** | 飞吻 | 捂嘴 | 口白 | — |
| **扎啤** | 砸 | 捂头 | 蹦床 | 金刚帽(对标氧气罐) |
| **电话** | 喂 | 捂耳朵 | 挂电话(→**超挂电话**，见 §3.2) | — |

- **反弹克制**把对应通道的攻击**弹回攻击者**（像反弹盾之于物理）；正因口红/扎啤/电话有此反弹（攻击有被弹风险），它们才比**无反弹的屁**弱。
- 每通道还各有**自己的运气动作**（名称待补；屁的是「上气」）。
- 通道全集 ＝ **物理**(波/刀/枪·反弹＝反弹盾) ＋ **4 魔法**(屁/口红/扎啤/电话)。

### 3.2 光轮（特殊运气；补全 / 放大）

**光轮**是一种特殊运气，积攒「光轮」点数；两种用法，用时消耗相关光轮：

**① 进攻 — 把攻击气「补全成 4 的倍数」**
- 每 1 光轮把当前某攻击气**向上补全到下一个 4 的倍数**（<4 补到 4，多 1 光轮再到 8、12…）。
- 例：`1波+1光轮 → 4 → pass`；`2波+1光轮 → 4 → pass`（放完清零，故称「补全」）；`1屁+2光轮 → 下屁强度 8`（1→4→8）。
- **光轮不能单独攻击**：`1波+2光轮` 放 pass 消耗 `1波+1光轮`，剩 1 光轮，单光轮放不了 pass。
- 固定耗攻击（pass=4）只吃够数光轮；释放全部型（下屁）吃光参与的全部气与光轮。

**② 防御 — 生成「超 xxx」**（消耗**全部**光轮）
- 把光轮加到**学到的防御**上 → 超反弹盾 / 超挂电话 / 超瞬间（**没有超防御**——基础防不可超）。
- **防御极限 ×4×光轮数**，档位形状不变（极限 ＝ 有兑档则取兑值，否则取反/防值）：
  - 反弹盾 1反2兑3破（极限兑=2）→ 1光轮 `2×4=8` → **7反8兑9破**；2光轮 `2×4×2=16兑`。
  - 挂电话 1反2破（极限反=1，无兑）→ 1光轮 `1×4=4` → 反弹≤4（挂4个电话）、≥5破。
  - **超瞬间例外**：瞬间 全通道 7防8破，按 **8 的倍数**算（打破瞬间叫「追上了」）→ 1光轮 = **31防32破**（多光轮公式❓）。

> **状态空间注意**：光轮是**确定**资源，但与（可能因氧气罐不确定的）攻击气一起决定「能否放某招/强度」→ 仍在 `S`（可能气态集合）框架内按分支算，只是每分支多一维光轮。

### 3.3 100代 的克 / 兑网（专注于克的模式）

本模式**防御只能被「克」破**（全发等纯力量无效）。两记 500 气大招专破防御，但各被一记小招克死：

| 招 (气) | 对防御 | 被克(致死) | 被兑 |
|---------|--------|-----------|------|
| 神龙 (500) | **兑防御**(穿防) | 元气弹 | 气元斩 |
| 神凤 (500) | **破防** | 气元斩 | 元气弹 |

- 即：元气弹(10) **克神龙·兑神凤**；气元斩(25) **克神凤·兑神龙**。
- 妙处：**10 气元气弹能克死 500 气神龙** → 囤大招有被廉价克制反杀的风险，故「专注于克」。
- 全发 / 盖亚(50) / 超盖亚(100)：纯力量，对**防御者无效**；用于打非防御目标。效果细节❓。
- 引擎：克/兑关系是**数据**（招A 克/兑 招B）；「防御只能被克破」是**模式级规则开关**。

**终极技：魔幻贯穿炮**
- 魔/幻/贯/穿/炮 ＝ 5 记攻击，波耗 **700/600/500/400/300**；**每释放一记 → 攒 1 个同名气**。
- 集齐 魔+幻+贯+穿+炮 5 种气 → 放大招 **魔幻贯穿炮**：**破防 且 无法被克**（同时无视防御与克系统）。
- **逐组件可用波替代**：缺哪种同名气，就用该记的波耗补 → 全用波 ＝ 700+600+500+400+300 ＝ **2500 波** 直接放。
- 引擎补充：① 5 种「同名气」是**攻击的副产物**（非运气获得）；② `cost` 新形态——**逐组件可替代**（同名气 或 波）；③ 技能可带**免疫标志**（无法被克 / 免疫破盾）。

**★ 核心机制：封印（control / 持续状态）**
- **单层**：消耗 100 波，封印**全场 3 回合**，被**小扫**克。
- **双层**：消耗 200 波，封印**全场 4 回合**，被**全扫**克。（**没有三层**）
- 封印是攻击技，但 **不致死、不能被防御**（纯控制）。
- **封印状态下的合法动作**（用别的 ＝ 溶）：
  - **封印者**：波 / 空 / 小扫 / 全扫 / 防御
  - **被封印者**：仅 防御
- 引擎补充：封印 ＝ **多回合持续状态**，按角色**限制合法动作集** ＋ **剩余回合**倒计时；
  **溶推广**为「用了状态不允许的动作」也判死；攻击效果不止伤害，还能**施加状态**。

## 4. 模式（每个模式 ＝ 可用技能集 ＋ 参数覆盖）

- **升级**（🔰 游戏最原始的版本，技能体系的骨架）
  - 玩法：2 人决斗，**赢家解锁下一个新技能**，按固定顺序逐步变强。
  - 技能解锁顺序：
    `波 → 刀 → 枪 → 气 → 来一点 → 反弹盾 → 口红 → 扎啤 → 电话 → 小霸 → 大霸 → 激光 → 光轮 → 瞬间`
  - **开局**：只能用 **波（运气）＋ 防御**。
  - 核心机制：**「成功攒到某种气，就解锁该种气的全部攻击方式」**——
    攻击不是单独解锁的，而是被「有没有对应的气」门控（再叠加气量是否够 → 否则溶）。
  - **点波 / 推波 / 削波 默认禁用**（超模）→ 数据模型需 per-mode 的 `enabled` 开关。
  - **合击技**：集齐多种气后解锁——火龙(波+刀)、火凤(波+刀+枪)、刀枪不入(刀+枪)。
  - **代的循环 (prestige)**：走到顺序末尾「瞬间」后再赢一次 → **进位到「2代波」**：技能全部重新锁定、只剩波，
    但运气一次 +2。如此 `1代[波→…→瞬间] → 2代[波→…→瞬间] → 3代…` 循环；代越高，运气/吸取越快。
    （注：「100代」是**独立模式**、只是借用了高代数设定，规则见 §4 与 §3.3，与此 prestige 循环不同。）
  - 各技能细节见「技能总表」（逐个补充中）。
- **波决**（✅ 首个实现目标，规则已完整）
  - 人数：2 人（决）或多人围圈
  - 技能：波 / 防 / 空 / 小扫 / 全扫 / pass / 冲击波
  - 气只有「波」一种
- **100代**（🎯 专注于「克」的独立模式）
  - 只有「波」一种气；**初始即 100 代**，运一次 +100 波；每升级一次 **代 +100**（→ 200/300 代…）。
  - **关键规则：防御只能被「克」破**——纯力量（全发再多气）都破不了防，只有克关系能穿。
  - 额外技能：全发(发全部气) / 元气弹(10) / 气元斩(25) / 盖亚(50) / 超盖亚(100) / 神龙(500) / 神凤(500)。
  - 克/兑 网见 §3.3。
- **（大霸模式 / 气·捂鼻模式 / 点推削波扩展 / …）** ❓：待补

> 待补：每个模式叫什么名字、几人玩、有哪些可用技能、有无参数改动。

## 5. 术语表

- **溶**：气不够硬放招 / 放不存在的招 → 判死，且攻击不生效。
- **克**：某些攻击专门克制某些出招（结算第二步）。
- **兑**：半挡半不挡，清空防御方的气。
- **夹死**：3 人局里两人攻击把中间运气的人夹死。
- **明牌 / tell**：暴露意图的准备动作（如神龟）。
- **通道 (channel)**：攻击的种类（波 / 气 / …），防御按通道匹配。
- **神龟** ❓：冲击波运第 6 气的明牌动作名（记忆不确定）。
- **追上了**：打破「瞬间」防御。
- **代 / prestige**：走完一轮技能即进位下一代，重置解锁、放大运气/吸取。
- **超 xxx**：用光轮强化的「学到的防御」（超反弹盾 / 超挂电话 / 超瞬间；**无超防御**）。
- **互攻强度 / 破防强度**：攻击在「对拼」/「对防御」时各自的强度，均可独立于气耗。

## 7. 引擎设计要点（`bo_rules` 的实现锚点）

把以上规则蒸馏成「引擎必须装下的能力」——这是写实现 spec 的依据。
`bo_rules` 是**确定性纯函数**：`resolve(状态, 各玩家暗选) → (结算事件, 新状态)`，可单元测试。

1. **气 ＝「可能气态集合」`S`**：每元素是 `Map<气类型, Qi>` 向量；氧气罐延迟债用分支表示；`Qi` 是**定点数**（小数、精确比较）。
2. **多气类型**：波/刀/枪/霸/屁/口红/扎啤/电话/激光/光轮…，各有自己的运气动作。
3. **通道（防御匹配轴）独立于气类型**：物理(波刀枪) / 屁 / 口红 / 扎啤 / 电话 / 激光…；每通道有对应防御。
4. **代 ＝ per-玩家倍率**：放大运气/吸取，不放大攻击与基础防御；「学到的」防御随代升；prestige 循环（走完一轮 → 进位重置）。
5. **攻击有多个独立强度**：气耗(付费) / 破防强度(vs 防御；可破盾、可被特定防御免疫) / 互攻强度(vs 攻击；激光=3)。
6. **防御档位 per-技能配置**：`X防 / X反(反弹) / X兑 / X破`；可无兑档、可无限、可全通道、可多通道。
7. **`cost` 四形态**：固定量 / 全部某类型 / 任选 1 种（→ 延迟债分支） / **逐组件可替代**（魔幻贯穿炮：同名气 或 波）。部分气是**攻击副产物**（魔幻贯穿炮）而非运气获得。
8. **合击技**：跨气类型组合消耗（火龙/火凤/刀枪不入）。
9. **光轮**：进攻补全到 4 的倍数；防御生成超 xxx（极限 ×4×光轮；瞬间按 ×8）。
10. **特殊机制**：taunt（来一点/反弹盾抬高被瞄准优先级）、窃取（吸）、克/兑关系网、明牌（神龟）、AOE（下屁/小扫）、多目标（冲击波）、**免疫标志**（无法被克 / 免疫破盾）、**持续状态**（封印：多回合限制合法动作）。攻击效果不止伤害（可施加状态 / 不致死 / 不可防）。
11. **结算管线**：先判溶（资源不足 **或** 违反状态限制）→ 再判克（目标优先级 / 互攻 / 攻防档位 / 反弹 / 兑清气）→ 施加/递减**持续状态**（封印）→ 出局重开（幸存者清气）→ prestige。
12. **全数据驱动**：技能 / 模式（可用技能集 + `enabled` + 参数 + **规则开关**，如「防御只能被克破」）/ 代 / **克兑关系** 都是数据；引擎逻辑与数据分离。
